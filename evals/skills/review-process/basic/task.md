# Eval: review-process-basic

## Task
Review the code changes in the provided branch or pull request.
Ensure that:
1. You identify and catalog findings using severity levels: Critical, Major, Minor, Nitpick.
2. You structure findings in a summary table grouped by severity.
3. You verify that the changeset includes appropriate tests for all new/modified logic.
4. You check for correctness, security, safety, and domain language alignment.
5. You generate a clear self-review checklist for the author.

## Success Criteria
- A structured review report is presented.
- All major/critical issues (if any) are flagged.
- A self-review checklist is provided.
