# Eval: test-planning-process-basic

## Task
Plan the test coverage for a new payment processing service:
1. Define the test boundaries (unit vs request/integration).
2. Create a test matrix including happy paths, boundary conditions, edge cases, and expected error flows.
3. Identify the very first failing test case to write before writing any implementation code.
4. Ensure synthetic test data is used.

## Success Criteria
- A detailed test plan/matrix document is created.
- The test boundaries and first failing test are clearly specified.
- The plan covers both success and error branches.
